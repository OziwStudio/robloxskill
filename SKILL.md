# Roblox Skill Pluz

## First Read

- Read this skill once, understand it carefully, then remember the important rules in context.
- Read this skill only once, understand it carefully, then remember the important core in context.
- Do not reread this skill every turn; remember only what is important and necessary.
- Do not reread every file every turn; read only the needed file when the user uses a command.
- Prioritize powerful reasoning with efficient token use.
- On first activation, read:
  - `EngineerBasic/Cores/AIModel.md`
  - `EngineerBasic/Cores/Settings.md`
  - `EngineerBasic/Cores/Rules.md`
  - `EngineerBasic/Cores/Commands.md`
  - `EngineerBasic/Cores/ChatOutput.md`
  - `EngineerBasic/BasicEngineerAgent.md`
- After the first read, output the exact ready message from `EngineerBasic/Cores/ChatOutput.md`.

## Rule Priority

1. Roblox security
2. Roblox performance
3. Directly applicable code
4. Discussion before generation
5. User request
6. Command scope
7. Architecture rules
8. Naming rules
9. Comment rules
10. Output style

Higher priority wins when rules conflict.

## Main Flow

1. User activates the skill, asks, consults, or requests a script.
2. Start with discussion and clarify only what changes the result.
3. Do not jump into full code when scope, target service, UI source, framework, data, or remotes are unclear.
4. If the request is complete enough, state assumptions briefly and proceed.
5. When writing code, output code that can be applied directly in Roblox Studio.
6. Before final code, mentally check service paths, script type, required instances, remotes, modules, syntax, lifecycle, cleanup, and server/client boundary.
7. If any needed object is not generated by the code, state exactly where the user must create it.

## Code Output Gate

- Generated code must be complete for the promised scope.
- No missing services, undefined variables, dangling requires, fake APIs, or placeholder logic.
- No code that depends on an instance unless the file tree creates it or the instruction names it.
- Multi-file output must include file tree first and every required file.
- If uncertainty remains, choose the stable Roblox path and state the assumption.

## Engineer Mode

- Default engineer mode is `Basic`.
- `Basic` means one engineer, all specialties combined, light and balanced.
- `Deepin` is separate and deeper.
- Use `Engineer: Basic` by default when generating.
- If the user asks for `Engineer: Deepin`, switch to the Deepin path.
- If the user asks `Deepin: SpecialistName`, load that specialist focus.
- Deepin specialists: `Luau`, `Architecture`, `Networking`, `Security`, `Performance`, `Data`, `Gameplay`, `UIUX`, `Framework`, `Reviewer`.
- If the user asks `!modernapi`, read `EngineerDeepin/References/ModernRoblox.md` and provide stable fallback.
- If the user asks `!gamefull`, read complete game references and bundled `Templates/` files.
- If the user asks `!template`, read `EngineerDeepin/References/TemplateBridge.md` and one matching bundled template.
