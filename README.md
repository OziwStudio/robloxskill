# Terla Community Roblox Skill

## What This Is

`Roblox Skill Pluz` is a Roblox Studio and Luau skill built for fast, structured, production-minded work. It supports light generation, deeper specialist analysis, bundled templates, community safety, animation/VFX, and modern Roblox guidance while keeping token use lean.

Core principle: `SUPER POWER TAPI HEMAT TOKEN`.

## Identity

- Community: Terla Community
- Author: The Owner (Oziwwwwww)
- Skill name: `Roblox Skill Pluz`
- Default engineer: `Basic`
- Version: `1.3.0`
- Primary language: English
- Secondary language: Indonesian

## How It Behaves

By default, the skill stays light, reads once, and keeps only the important rules in context. It does not pull extra support unless you enable it.

## Optional Support

- `Management Project` for Rojo-style project workflow and tooling
- `Framework` for Knit, ProfileStore, Fusion, and related architecture

## Engineer Mode

`Basic` is the default engineer. It is balanced, practical, and token-efficient for normal Roblox work.

`Deepin` is the principal-depth analysis mode. It adds 10 specialist domains, risk matrix routing, stronger review gates, modern Roblox guidance, complete game flow, QA, LiveOps, monetization safety, Studio mode routing, and a bundled `Templates/` bridge.

## Ready Assets

- Genre templates are included and ready in `Templates/`.
- Deepin specialist files are included in `EngineerDeepin/Specialists/`.
- Deepin has manifest files for template and specialist completeness.
- Deepin has concrete implementation patterns for modules, cleanup, remotes, validation, data schema, receipts, combat, and round flow.
- Modern Roblox guidance includes 2026-style API confidence gates and stable fallback rules.
- Deepin includes dedicated Animation/VFX guidance for particles, tweens, camera shake, SFX, cutscenes, and gameplay feedback.
- Deepin includes mesh fidelity and LOD guidance for `RenderFidelity`, `CollisionFidelity`, and `LevelOfDetail`.
- Deepin includes community safety 2026 guidance for chat, UGC, ads, and policy-sensitive systems.

## Flow

1. Read the skill once.
2. Start in `Basic` unless `Deepin` is requested.
3. Load only the files needed for the current task.
4. Enable support only when the user asks for it.
5. Use `Deepin` for audits, large systems, risky work, or specialist depth.

## Why It Stands Out

- Two-engineer model with clear depth control.
- Basic mode stays short and efficient.
- Deepin mode adds specialist routing without forcing every file into memory.
- Deepin uses principal engineer doctrine, depth levels, risk matrix routing, and synthesis gates.
- Optional support keeps frameworks and project tooling out of the way unless needed.
- Template bridge lets complete game work stay reusable instead of repetitive.
- 2026 API handling is explicit: verify-current APIs need fallback or user approval.
- Animation/VFX is first-class, not only a performance footnote.
- Community safety is first-class for social, chat, UGC, ads, and youth-audience systems.
